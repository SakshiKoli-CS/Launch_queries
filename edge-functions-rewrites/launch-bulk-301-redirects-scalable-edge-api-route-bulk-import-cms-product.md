QUESTION

Customer **Horizontal** (SF case **00050147**; Next.js 15 App Router on Launch) has **~396 301 redirects** and wants a **scalable/automated way to manage them** — ideally **bulk import/export**, or edge integration **without custom scripting** — rather than adding each redirect entry manually in Contentstack (per the [redirecting-urls guide](https://www.contentstack.com/docs/developers/how-to-guides/redirecting-urls)). Is there a **built-in/automated solution for a large number of 301 redirects**, and specifically any **utility/plugin/feature for bulk import/export of redirect entries in the CMS**?

_Keywords: manage many 301 redirects, 396 redirect entries, bulk import export redirects, scalable redirects without manual entry, edge function redirects at scale, API route s-maxage cached redirects, launch.json redirects, redirect import/export CMS product team not Launch._

ANSWER

## Scalable redirects on Launch — edge + cached API route (avoids per-redirect manual work)

- Recommended scalable pattern: a **Next.js API route** (e.g. `/pages/api/redirects`) that **fetches redirect entries from the CMS** with **`Cache-Control` (`s-maxage`)** so they're cached at the edge/CDN, read by a **Launch Edge Function** (`/functions/[proxy].edge.js`) that applies redirects at the edge:
  ```js
  const response = await fetch(new Request(apiUrl, request)); // preserves original request headers/context
  ```
- This gives **automation + performance** — no per-redirect manual setup, redirects managed via the CMS, applied at the edge. Example: **`contentstack-launch-examples/launch-edge-function-redirect`**.
- (Redirects can also live in **`launch.json`** at the edge; a large list can be **generated by a script** from source data — see the CMS-driven / build-time-generation entries.)

## Bulk import/export of redirect entries — that's a CMS (Product) topic, not Launch

- The specific ask — a **utility/plugin/built-in feature to bulk import/export redirect *entries* within the CMS** — **isn't a Launch concern.** It's about **Contentstack CMS features**, so it should go to the **CMS Product team** for guidance on bulk entry import/export utilities/plugins.
- **Distinguish the two questions:** *how redirects are applied* (Launch: edge function / `launch.json` / cached API route) vs *how redirect entries are bulk-loaded into the CMS* (CMS import/export — Product).

## Generalized guidance (for future similar queries)

- **"Scalable way to manage many (hundreds of) 301 redirects on Launch?"** → **edge function reading a cached API route** (CMS-sourced, `s-maxage`) or **`launch.json`** (optionally **script-generated** from source data). No per-redirect manual coding. See `launch-edge-function-redirect`.
- **"Bulk import/export of redirect entries?"** → that's a **CMS/Product** capability (entry import/export), **not Launch** — route to the Product team.
- Related: [CMS-driven redirects no-redeploy/no-per-request (cached API route)](launch-cms-driven-redirects-no-redeploy-no-per-request-api-edge-api-route-cache.md), [domain migration redirects via edge function + launch.json](launch-domain-migration-redirect-old-domains-to-subfolders-edge-function-domain-limit.md), [edge function deploy fails / malformed env var](launch-edge-function-deploy-fails-one-env-malformed-env-var-quotes.md).
